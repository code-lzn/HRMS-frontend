# 2. 详细设计

## 2.1 前端迭代目标

1. 部门树管理：左侧部门树 + 右侧部门详情，支持新增/编辑/删除/合并
2. 部门详情直入：通过 URL 参数 `?deptId=7` 直接加载单个部门详情
3. 部门人数统计：实时显示各部门在职人数（含子部门递归汇总）
4. 职位管理：职位列表页 + 新增/编辑职位表单
5. 序列职级对照：序列-职级对照表展示

## 2.2 迭代具体描述

### 2.2.1 部门树管理页

##### UI&交互

- 左右分栏布局：左侧部门树（可展开/折叠），右侧部门详情 + 下级部门列表
- 点击树节点切换选中部门，右侧联动更新（直接从已有树数据取，无需额外请求）
- 通过 URL 参数 `?deptId=7` 直达时，调用 `/detail` 接口获取单个部门详情
- 部门树支持搜索过滤
- 每层节点显示「在职人数」badge

##### 前端逻辑

###### 页面布局

- 面包屑导航：首页 > 组织架构 > 部门管理
- 搜索框：可搜索部门名称，在树中高亮匹配节点
- 工具栏按钮：新增部门、合并部门

###### 部门树组件

| 特性 | 说明 |
| --- | --- |
| 数据加载 | 进入页面一次性加载完整部门树 `GET /api/departments/tree` |
| 节点展示 | 部门名称 + 在职人数 badge（在职 + 试用期员工） |
| 展开/折叠 | 默认展开前2级，其他折叠 |
| 选中态 | 蓝色高亮，右侧显示该部门详情 |
| 最大层级 | 5级，第5级节点的新增按钮隐藏 |
| 空状态 | 显示"暂无部门数据"占位图 |
| 孤儿节点 | 父部门被删除的节点自动归入根层级显示 |

###### 右侧部门详情面板

展示选中部门的信息：
- 部门名称、部门编码（只读）、上级部门、部门负责人、排序序号、部门描述
- 下级部门列表（表格）：部门名称、编码、负责人、在职人数、排序、操作（编辑/删除）
- 「编辑」按钮 → 弹出编辑弹窗（回填当前部门数据）
- 「删除」按钮 → 二次确认 → 后端校验无子部门+无在职员工 → 删除

###### 新增部门弹窗

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 最大输入长度 | 输入限制 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 部门名称 | 如"技术部" | Input | Y | - | 32 | - | string | 请输入部门名称 | - |
| 部门编码 | 2位，用于工号 | Input | Y | - | 2 | 字母或数字 | string | 请输入2位编码 | - |
| 上级部门 | 默认为当前选中部门 | TreeSelect | N | 选中部门 | - | - | number | 请选择上级部门 | 部门树 |
| 部门负责人 | - | Select + 搜索 | N | - | - | - | number | 请选择负责人 | 员工列表接口 |
| 排序序号 | 越小越靠前 | InputNumber | Y | 0 | - | 正整数 | number | - | - |
| 部门描述 | 职能说明 | TextArea | N | - | 128 | - | string | 请输入部门描述 | - |

**前端需拦截的校验**（后端也有兜底）：
- 部门名称不能为空
- 部门编码不能为空
- 当前层级的兄弟部门名称不可重复（需前端遍历同级节点判断，减少无效请求）

###### 编辑部门弹窗

| 字段名称 | 说明 | 输入方式 | 是否必填 | 说明 |
| --- | --- | --- | --- | --- |
| 部门名称 | 回填当前值 | Input | Y | 不能为空 |
| 部门编码 | 回填当前值 | - | - | **只读**，创建后不可修改 |
| 上级部门 | 回填当前值 | - | - | **只读**，变更需走合并流程 |
| 部门负责人 | 回填当前值 | Select + 搜索 | N | - |
| 排序序号 | 回填当前值 | InputNumber | Y | - |
| 部门描述 | 回填当前值 | TextArea | N | - |

###### 合并部门弹窗

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 源部门 | 被合并 | Select（排除当前选中部门） | Y | - | number | 请选择源部门 | 部门树下拉 |
| 目标部门 | 保留 | Select | Y | - | number | 请选择目标部门 | 部门树下拉 |

校验：
- 源部门 ≠ 目标部门
- 目标部门不能是源部门的子部门
- 合并前展示"将转移 X 名员工和 Y 个子部门，确定合并？"（合并成功后根据返回的 `transferredEmployees` 和 `transferredChildDepts` 提示）

###### 操作按钮

| 按钮 | 交互 | 二次确认 | 显示/禁用控制 |
| --- | --- | --- | --- |
| 新增部门 | 打开新增弹窗 | N | 需有 `org:manage` 权限（roleId 1/2） |
| 编辑部门 | 打开编辑弹窗 | N | 需有 `org:manage` 权限（roleId 1/2） |
| 删除部门 | 调用删除接口 | Y，"确定删除该部门吗？" | 需有 `org:manage` 权限（roleId 1/2） |
| 合并部门 | 打开合并弹窗 | Y，展示转移统计 | 需有 `org:manage` 权限（roleId 1/2） |

> 无 `org:manage` 权限的用户（roleId 3/4/5）仍可查看部门树和基本信息，但所有操作按钮隐藏。

##### 所需API

| 接口 | 说明 | 请求方式 |
| --- | --- | --- |
| `GET /api/departments/tree` | 获取部门树（含递归人数） | 无参数 |
| `GET /api/departments/detail?id={id}` | 查询单个部门详情 | Query String |
| `POST /api/departments/add` | 新增部门 | JSON Body |
| `PUT /api/departments/update` | 编辑部门 | JSON Body（id 在请求体） |
| `POST /api/departments/delete` | 删除部门 | JSON Body `{ "id": xx }` |
| `POST /api/departments/merge` | 合并部门 | JSON Body |

### 2.2.2 职位列表页

##### UI&交互

- 顶部筛选栏 + 表格列表
- 序列用 Tab 切换（全部 / M管理 / P专业 / S支持）
- 表格列：职位名称、序列、所属部门、职级范围、默认试用期、操作

##### 前端逻辑

###### 列表搜索字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 序列 | 职位序列 | Tab 切换 | N | 全部 | string | - | 枚举：M/P/S |
| 所属部门 | 按部门过滤 | TreeSelect | N | 全部 | number | 请选择部门 | 部门树 |

###### 列表展示字段

| 字段名称 | 说明 |
| --- | --- |
| 职位名称 | 如"Java开发工程师" |
| 序列 | M管理 / P专业 / S支持 |
| 所属部门 | 为空显示"全公司通用" |
| 职级范围 | 如"P1-P10"（接口返回 `levelRange` 字段） |
| 默认试用期 | 如"3个月"（接口返回 `defaultProbationMonths`） |
| 职位描述 | 超长截断 + Tooltip |

###### 列表工具栏

| 按钮 | 交互 | 显示控制 |
| --- | --- | --- |
| 新增职位 | 打开新增弹窗 | 需有 `org:manage` 权限（roleId 1/2） |
| 序列职级对照 | 打开对照表抽屉 | 始终显示 |

###### 操作按钮

| 按钮 | 交互 | 二次确认 | 显示控制 |
| --- | --- | --- | --- |
| 编辑 | 打开编辑弹窗 | N | 需有 `org:manage` 权限（roleId 1/2） |
| 删除 | 调用删除接口 | Y，"确定删除该职位吗？" | 需有 `org:manage` 权限（roleId 1/2） |

> 无 `org:manage` 权限的用户（roleId 3/4/5）仍可查看职位列表，但所有操作按钮隐藏。

##### 所需API

| 接口 | 说明 | 请求方式 |
| --- | --- | --- |
| `GET /api/positions/list?sequence={x}&departmentId={x}` | 获取职位列表 | Query String |
| `POST /api/positions/add` | 新增职位 | JSON Body |
| `PUT /api/positions/update` | 编辑职位 | JSON Body（id 在请求体） |
| `POST /api/positions/delete` | 删除职位 | JSON Body `{ "id": xx }` |
| `GET /api/positions/sequences` | 获取序列职级对照 | 无参数 |

### 2.2.3 职位新增/编辑表单

##### 表单字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 最大输入长度 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 职位名称 | 如"Java开发工程师" | Input | Y | - | 32 | string | 请输入职位名称 | - |
| 职位序列 | M/P/S | Radio | Y | P | - | string | - | 枚举 |
| 所属部门 | 空=全公司通用 | TreeSelect | N | - | - | number | 请选择部门 | 部门树 |
| 职级下限 | 如P1 | Cascader | Y | - | 8 | string | 请选择 | `/api/positions/sequences` |
| 职级上限 | 如P10 | Cascader | Y | - | 8 | string | 请选择 | `/api/positions/sequences` |
| 默认试用期(月) | 发起入职时自动填充 | InputNumber | Y | 3 | - | number(1-12) | - | - |
| 职位描述 | 岗位职责说明 | TextArea | N | - | 256 | string | 请输入职位描述 | - |

**前端需拦截的校验**（后端也有兜底）：
- 职位名称不能为空
- 职级下限不能高于职级上限（需遍历 `/sequences` 返回的 `levels` 数组比较 index）
- 选择序列后，职级下拉选项从对应序列的 `levels` 数组中联动过滤

### 2.2.4 序列职级对照表

##### UI

- 抽屉 / Modal 展示
- 三列卡片布局：管理序列 M（M1-M5）、专业序列 P（P1-P10）、支持序列 S（S1-S5）

##### 数据源

```
GET /api/positions/sequences
```

**响应格式**：
```json
{
  "code": 0,
  "data": [
    { "sequence": 1, "sequenceName": "管理序列", "sequenceCode": "M",
      "levels": ["M1", "M2", "M3", "M4", "M5"] },
    { "sequence": 2, "sequenceName": "专业序列", "sequenceCode": "P",
      "levels": ["P1", "P2", "P3", "P4", "P5", "P6", "P7", "P8", "P9", "P10"] },
    { "sequence": 3, "sequenceName": "支持序列", "sequenceCode": "S",
      "levels": ["S1", "S2", "S3", "S4", "S5"] }
  ],
  "message": "ok"
}
```

## 5.1 风险评估

- 部门合并涉及员工批量转移，前端需展示准确的转移统计数据（接口返回 `transferredEmployees` 和 `transferredChildDepts`）
- 部门树大数据量（500+节点）时需考虑虚拟滚动优化
- 编码创建后不可回收复用，若用户输错编码需用新编码重试
