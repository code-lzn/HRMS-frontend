### 2.2.2 我的考勤

##### UI

- **顶部打卡区**（工作日显示）：
  - 当前日期 + 实时时钟
  - 上班打卡按钮 / 下班打卡按钮（状态互斥）
  - 已打卡显示时间 + 判定结果（正常/迟到/早退）

- **月度统计卡片**：
  - 正常出勤 / 迟到 / 请假 / 缺卡天数

- **考勤日历**：
  - AntD Calendar 组件
  - 日期格子颜色标记：绿=正常 / 橙=迟到早退 / 红=旷工 / 蓝=请假 / 灰=缺卡
  - 点击日期 Popover 显示详细打卡记录

- **快捷操作**：
  - 「申请请假」按钮 → 跳转我的请假
  - 「申请补卡」按钮 → 针对缺卡日期

##### 打卡交互

```typescript
// 上班打卡
POST /attendance/punch  { punchType: 0, location: "..." }
// 下班打卡
POST /attendance/punch  { punchType: 1, location: "..." }
```

##### 所需API

| 接口 | 说明 |
| --- | --- |
| POST /attendance/punch | 上班/下班打卡 |
| GET /attendance/today | 获取今日打卡状态 |
| GET /attendance/calendar?month=YYYY-MM | 考勤日历 |
| GET /attendance/records?month=YYYY-MM | 当月考勤记录列表 |

##### 考勤日历响应结构

```json
{
  "month": "2026-07",
  "normalDays": 5,
  "lateDays": 0,
  "leaveDays": 0,
  "missingDays": 0,
  "dailyStatus": {
    "2026-07-03": 2,
    "2026-07-06": 0
  },
  "makeupAvailableDates": []
}
```

### 2.2.3 我的请假

##### UI

- **请假列表**：表格展示，按时间倒序
- **「申请请假」按钮** → 跳转请假表单页（或 Modal）
- 列表展示：请假类型(Tag) + 起止日期 + 天数 + 状态(Tag) + 操作

##### 请假申请表单

| 字段 | 输入方式 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 请假类型 | Select | Y | 事假/病假/年假/婚假/产假/丧假/调休 |
| 开始日期 | DatePicker | Y | |
| 结束日期 | DatePicker | Y | >= 开始日期 |
| 原因 | TextArea | Y | |

##### 审批进度查看

点击列表项 → 进入详情页，底部显示审批进度时间线 (Timeline 组件)：

```
○ 提交申请 — 张三 · 2026-07-10 09:00
   原因：家庭事务

● 审批 — 进行中...
```

或：

```
○ 提交申请 — 张三 · 2026-07-10 09:00

○ 审批 — 李四 · 2026-07-10 14:30
   审批意见：同意
```

**节点状态与颜色**:

| status | 含义 | 图标颜色 |
| --- | --- | --- |
| 0 | 已完成 | 绿色 |
| 1 | 进行中 | 蓝色 |
| 2 | 未开始/已跳过 | 灰色 |

##### 取消操作

- 仅 `status=0`（待审批）时显示取消按钮
- 二次确认弹窗

##### 所需API

| 接口 | 说明 |
| --- | --- |
| POST /attendance/leave/apply | 申请请假 |
| GET /attendance/leave/my | 我的请假列表 |
| GET /attendance/leave/{id}/progress | 审批进度 |
| POST /attendance/leave/cancel/{id} | 取消申请 |

### 2.2.4 我的薪资

##### UI

- **工资条列表**：
  - 按月倒序展示
  - 每项：月份 + 应发 + 应扣 + 实发 + 状态 Tag
  - 点击某条 → 弹出密码验证 Modal

- **工资条详情 Modal**：
  - 首次查看 → 输入登录密码验证
  - 验证通过 → 展示仿真实工资条样式
  - 收入区（基本工资/津贴/绩效/加班费）+ 扣除区（社保/公积金/个税）+ 实发金额

- **薪资趋势图**：
  - AntV 折线图：近6个月应发工资 + 实发工资
  - 数据源：GET /salary/trend

##### 二次验证弹窗

| 字段 | 说明 |
| --- | --- |
| 密码 | 输入登录密码 |
| 确认 | 提交验证 → POST /salary/slip/{id} |

- 验证失败 → 提示"密码验证失败"
- 验证通过 → 展示工资条详情

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /salary/slips | 工资条列表 |
| POST /salary/slip/{id} | 查看工资条详情（body: { password }） |
| GET /salary/trend | 近6个月薪资趋势 |

##### 工资条详情响应字段说明

**收入项**: baseSalary(基本工资), allowance(岗位津贴), performanceBonus(绩效奖金), overtimePay(加班费), manualAdjust(手动调整)
**扣除项**: lateDeduction(迟到扣款), leaveDeduction(请假扣款), socialPension(养老保险), socialMedical(医疗保险), socialUnemployment(失业保险), housingFund(住房公积金), incomeTax(个人所得税)
**汇总**: grossSalary(应发), totalDeduction(应扣), netSalary(实发)

##### 薪资趋势响应结构

```json
[
  { "month": "2026-02", "netSalary": 11500.00, "grossSalary": 14800.00 },
  { "month": "2026-03", "netSalary": 11800.00, "grossSalary": 15000.00 }
]
```

> 数据已按时间正序排列，可直接绑定折线图。

### 2.2.5 账号安全

##### 修改密码

| 字段名称 | 输入方式 | 是否必填 | 校验规则 |
| --- | --- | --- | --- |
| 旧密码 | Password Input | Y | - |
| 新密码 | Password Input | Y | >=8位 |
| 确认新密码 | Password Input | Y | 与新密码一致 |

- 提交: `POST /account/changePassword`
- 错误提示映射：
  - 50018 → "原密码错误"
  - 50019 → "新密码不能与旧密码相同"
  - 50020 → "新密码与近期使用过的密码重复"
- 成功后 Toast "密码修改成功"

##### 绑定/修改手机号

| 字段名称 | 输入方式 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 手机号 | Input | Y | 传空字符串 = 解绑 |

- 提交: `POST /account/bindPhone` { phone: "138..." }
- 格式校验：`^1[3-9]\d{9}$`
- 错误码 50021 → 提示"该手机号已被其他账号绑定"

##### 登录日志

- 表格分页展示，最多30条
- 列：登录时间 + IP + 设备 + 登录方式 + 是否成功(绿勾/红叉) + 失败原因
- 仅支持查看，不支持删除

##### 所需API

| 接口 | 说明 |
| --- | --- |
| POST /account/changePassword | 修改密码 |
| POST /account/bindPhone | 绑定/解绑手机号 |
| GET /account/loginLogs | 登录日志（最近30条） |

# 5. 其他

## 5.1 风险评估

- 个人中心聚合多个模块，需确保各子模块数据一致性
- 工资条数据敏感，需确保前端不缓存到 localStorage
- 工资条每次查看都需要输入密码（未实现10分钟免验证），可能影响体验
- 当前未集成短信验证码服务，二次验证仅支持登录密码
