# 2. 详细设计

## 2.1 前端迭代目标

实现员工个人中心工作台，包括：

1. **我的档案**：查看个人基础信息（敏感字段脱敏），编辑可修改字段（邮箱、地址、紧急联系人），锁定字段灰显提示
2. **我的考勤**：日历视图展示当月出勤状态，网页端打卡（上班/下班），月考勤统计，补卡申请入口
3. **我的请假**：请假列表（含状态筛选），新建请假入口，审批进度查看，取消审批中申请
4. **我的薪资**：工资条列表（按月），工资条详情（二次验证：短信验证码/密码），薪资趋势折线图（AntV）
5. **账号安全**：修改密码，绑定/修改手机号，登录日志查看
6. **待办红点**：顶部导航待办数量角标

**技术栈：** React + TypeScript + Umi + Ant Design + AntV（折线图） + Zustand + umi-request + dayjs

**请求层封装：** 使用 UMI 内置的 `umi-request`（底层基于 axios），在 `app.ts` 的 `request` 配置中统一设置 baseURL、超时 30s、请求拦截器（自动注入 token）、响应拦截器（code≠0 统一 Toast + 401 跳转登录）。因个人中心所有接口仅操作本人数据，前端**不传 userId 参数**，由后端从 token 提取。各业务模块通过自定义 Hook 封装请求逻辑，返回 `{ data, loading, error, refresh }`。

##### 前端逻辑

###### 二次验证流程

1. 判断 `hasViewed` 是否为 false
2. 弹出验证弹窗，提供两种验证方式：
   - **短信验证码**：点击"发送验证码"→ 倒计时 60s → 输入 6 位验证码
   - **密码验证**：输入登录密码
3. 验证成功后，`hasViewed` 置为 true，展示工资条详情
4. 验证失败 3 次后锁定 15 分钟（后端控制 + 前端倒计时提示）

## 2.3 菜单与权限变动

### 2.3.1 菜单结构

```
个人中心 （所有角色可见，仅展示自己的数据）
├── 我的档案
├── 我的考勤

└── 账号安全
```

### 2.3.2 权限控制

- **数据隔离（核心）**：所有个人中心接口从 token 中提取 userId，前端无需传 userId 参数，**禁止通过参数指定用户ID**（防止水平越权）
- 所有角色均可访问个人中心，但仅能查看/操作自己的数据
- 敏感字段脱敏在后端处理，前端直接展示后端返回的脱敏数据
- 锁定字段由后端 `editableFields` / `lockedFields` 控制，前端据此渲染

# 5. 其他

## 5.1 风险评估

| **风险项**                         | **风险等级** | **应对措施**                                                 |
| ---------------------------------- | ------------ | ------------------------------------------------------------ |
| 水平越权（查看他人数据）           | 高           | 前端不传 userId，所有接口从 token 提取用户身份                |
| 工资条二次验证短信不可用           | 中           | 降级方案：支持密码验证替代短信验证码                         |
| 打卡并发（快速双击）               | 低           | 按钮点击后立即 disabled，接口返回后恢复                      |
| 考勤日历跨月数据加载               | 低           | 月份切换时重新请求，loading 态友好提示                        |
| 敏感字段在前端意外暴露             | 高           | 前端不自行处理脱敏，严格展示后端返回的脱敏数据               |
| 修改密码后 token 未失效导致继续操作 | 中           | 密码修改成功后前端清除 token，跳转登录页                      |

## 5.2 稳定性保障

- **Token失效处理**：密码修改后前端清除 localStorage token + 跳转登录页
- **打卡防重**：Redis 分布式锁 + 数据库唯一约束（employee_id + date）+ 前端 disabled
- **敏感数据**：一律使用后端脱敏后的数据，前端不持有原始敏感数据
- **错误处理**：验证码错误、验证次数超限等场景友好提示
- **兼容性**：Chrome 90+ / Firefox 88+ / Edge 90+ / Safari 14+，分辨率 1366×768+，移动端打卡需支持响应式
