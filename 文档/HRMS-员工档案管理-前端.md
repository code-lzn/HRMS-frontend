# HRMS-员工档案管理-前端

## 需求背景

本模块来源于 HRMS（人资管理系统）产品规格说明书中第 4 部分——员工档案管理。当前公司内部员工档案依赖 Excel 与纸质管理，存在数据分散、更新不及时、权限控制粗放等问题。本模块旨在实现员工档案的数字化统一管理，涵盖员工列表查询、档案详情展示、档案编辑等功能，同时为入转调离、薪资核算、考勤管理等下游模块提供基础数据支撑。

> **说明**：新建员工档案页面归入入职流程模块，调岗、离职操作在列表页提供入口，具体页面归入入转调离模块。本模块聚焦于员工档案的查询、查看和编辑。

## 详细设计

### 前端迭代目标

本次前端开发聚焦于员工档案管理的三大页面：

1. **员工列表页**：高级搜索（关键词/部门/职位/在职状态/职级/入职日期范围）、表格展示、导出 Excel、操作入口（查看详情/编辑/调岗/离职）
2. **员工档案详情页**：按信息分组（基础信息、个人信息、工作信息、薪资与合同）卡片展示、敏感字段脱敏、变更历史查看
3. **员工档案编辑页**：按角色和字段权限控制可编辑项、锁定字段提示、表单校验、仅提交变更字段

> **核心难点**：字段级权限控制——不同角色对不同字段的查看、编辑权限不同，需通过接口动态获取权限配置并控制前端渲染。

#### 员工列表页

##### UI&交互

UED 地址：参见 PRD 文档中的员工列表原型图

整体布局采用 **搜索栏 + 表格 + 分页** 的标准管理页面布局。

**搜索栏交互：**

| **筛选项** | **组件** | **说明** |
| --- | --- | --- |
| 关键词 | Input + Search | 支持姓名/工号/手机号模糊匹配，防抖 300ms 后自动搜索 |
| 部门 | TreeSelect（多选） | 展示部门树，支持父子联动选择 |
| 职位 | Select（多选） | 下拉多选，支持搜索过滤 |
| 在职状态 | Checkbox.Group | 四个状态多选：试用期、正式、待离职、已离职 |
| 职级 | Select（多选） | 如 P5、P6、M2 等 |
| 入职日期范围 | DatePicker.RangePicker | 选择入职日期起止范围 |

**表格列交互：**

| **列** | **渲染方式** | **说明** |
| --- | --- | --- |
| 姓名 | 链接文本 | 点击跳转详情页 |
| 工号 | 等宽字体 | — |
| 部门 | 纯文本 | 超长省略 + Tooltip |
| 职位 | 纯文本 | 超长省略 + Tooltip |
| 职级 | 纯文本 | — |
| 在职状态 | `<Tag color="...">` | 试用期=蓝色、正式=绿色、待离职=橙色、已离职=灰色 |
| 入职日期 | 纯文本 | `YYYY-MM-DD`，支持列排序 |
| 操作 | Button + Dropdown | 查看详情 / 编辑 / 更多操作（调岗、离职） |

**操作列交互规则：**

- **查看详情**：跳转 `/employees/:id`，所有角色可见
- **编辑**：跳转 `/employees/:id/edit`，按角色控制——HR/管理员对全部员工可见，部门主管对本部门可见，普通员工仅对自己可见
- **调岗/离职**：仅当在职状态为"试用期"或"正式"时可用，否则置灰
- **普通员工**：仅能查看自己的记录

**导出交互：**

- 导出按钮位于页面右上角，仅 HR/管理员和部门主管可见
- 按当前搜索条件导出全量数据（不分页）为 Excel
- 导出过程按钮显示 loading，完成后触发浏览器下载
- 文件名格式：`员工档案_{yyyyMMdd}.xlsx`

**删除确认弹窗交互：**

（员工档案不支持直接删除，删除由入转调离模块的离职流程处理。）

```plain

```

**请求参数**

| **参数** | **类型** | **必填** | **描述** |
| --- | --- | --- | --- |
| keyword | String | 否 | 模糊搜索关键词（姓名/工号/手机号） |

    "total": 156,

        "id": 1001,

#### 员工档案详情页

##### UI&交互

UED 地址：参见 PRD 文档中的员工详情原型图

整体布局采用 **顶栏（返回 + 操作按钮） + 信息卡片组（栅格布局） + 底部变更历史** 结构。

**顶栏交互：**

| **元素** | **内容** |
| --- | --- |
| 返回按钮 | 点击跳转回列表页，恢复之前的搜索条件和页码 |
| 页面标题 | "员工档案详情" |
| 操作区 | HR/管理员可见编辑按钮；部门主管对本部门员工可见；普通员工仅对自己可见 |

**信息卡片分组：**

| **卡片** | **内容** | **说明** |
| --- | --- | --- |
| 基础信息 | 工号、系统账号、在职状态、入职日期、入职类型、创建时间 | `Descriptions` 组件展示 |
| 个人信息 | 姓名、性别、手机号、邮箱、身份证号、生日、户籍地址、现居住地址 | 敏感字段脱敏渲染 |
| 工作信息 | 所属部门、职位、职级、直接汇报人、工作地点 | 直接汇报人可点击跳转详情 |
| 薪资与合同 | 合同类型、到期日、试用比例、账套、基本工资、银行账号、开户行 | 无权限时整卡显示"无权限查看" |

**变更历史展示：**

- 详情页底部默认展示最近 3 条变更记录
- 每条记录：变更字段中文名、变更前值 → 变更后值、变更类型标签、操作人、时间
- 变更类型标签颜色：DIRECT_EDIT=蓝色、FLOW_CHANGE=橙色、SYSTEM=灰色
- 点击"查看全部"打开 Drawer，分页加载更多

**提交流程：**

```
用户点击「保存」
  → 表单校验 → 失败：字段级错误提示，自动滚动到第一个错误字段
  → 校验通过 → 收集已修改（dirty）的字段，仅提交变更字段
  → PUT /api/v1/employees/{id} → 按钮 loading + disabled
    → 成功：提示 "保存成功"，跳转详情页
    → 冲突(409)：Modal 提示 "数据已被他人修改，请刷新后重试"
    → 网络错误：提示 "保存失败，请重试"
```

**取消确认：**

- 点击「取消」或浏览器后退时，检测 `form.isFieldsTouched()`
- 有未保存修改 → `Modal.confirm`："您有未保存的修改，确定要离开吗？"
- 无修改 → 直接返回详情页

### 风险评估

| **风险项** | **影响** | **应对措施** |
| --- | --- | --- |
| 字段权限配置变更（角色、可见性调整） | 前端硬编码权限需发版更新 | 权限通过接口 `GET /api/v1/employees/field-permissions` 动态获取，后端集中管理，修改权限不需前端发版 |
| 敏感数据泄露（前端脱敏不彻底） | 数据安全 | 后端在 Service 层返回 VO 前完成脱敏；前端脱敏组件作为兜底，双重保障 |
| 大量员工数据列表渲染卡顿 | 用户体验 | TanStack Query 分页查询 + `keepPreviousData` 避免翻页闪烁；虚拟滚动作为后续优化项 |
| 角色权限定义变更 | 按钮可见性逻辑需调整 | 权限判断集中到统一的 `useAccess` hooks，避免分散在各组件中 |
| 浏览器兼容 | IE 不支持 | 明确仅支持 Chrome 90+ / Firefox 88+ / Edge 90+ / Safari 14+ |

### 稳定性保障

- **兼容性**：支持 Chrome 90+ / Firefox 88+ / Edge 90+ / Safari 14+
- **最低分辨率**：1366 × 768
- **数据刷新**：编辑保存成功后立即刷新详情和列表查询缓存，确保数据一致性
- **错误兜底**：所有 API 调用统一错误拦截——网络异常和业务错误码分别处理，页面级别 `ErrorBoundary` 兜底，避免白屏
- **空状态处理**：列表无数据时区分"暂无数据"与"无搜索结果"，分别展示引导文案和重置搜索入口
- **脱敏兜底**：后端脱敏 + 前端 `DesensitizedText` 组件双重保障，`hasPermission=false` 时显示 `****`
- **防重复提交**：保存按钮 loading 期间 `disabled`，导出按钮同理
