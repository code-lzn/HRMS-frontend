## 1. 项目背景

> 对本次项目的背景以及目标进行描述，方便开发者理解需求，对齐上下文。

### 1.1 业务背景

本模块来源于 HRMS（人资管理系统）产品规格说明书中第 9 部分——个人中心。当前系统缺少面向普通员工的统一自助服务入口，员工查看个人信息、考勤记录、工资条等需通过 HR 人工查询，效率低且体验差。本模块旨在为每位员工提供个人中心工作台，聚合展示个人档案、考勤、请假、薪资等核心信息，支持自主编辑可修改字段、申请请假/补卡、查看工资条、管理账号安全等常用操作，减少 HR 的事务性工作。

## 3. 功能模块

> 描述本次迭代涉及的功能与场景。

### 3.1 核心功能

1. **我的档案**：查看个人基础信息（敏感字段脱敏），编辑可修改字段（邮箱、现居住地址、紧急联系人等），锁定字段（部门、职位、薪资）展示但不可编辑
2. **我的考勤**：日历视图展示当月出勤状态（正常/迟到/早退/旷工/请假/缺卡），支持网页端打卡（如考勤组配置允许），缺卡日期提供补卡申请入口（跳转考勤模块，具体补卡审批流由考勤模块+审批中心处理）
3. **我的请假**：请假申请列表（含审批状态筛选），查看审批进度，取消审批中的请假申请
4. **我的薪资**：历史工资条列表（按月），查看工资条详情（首次查看需二次验证——短信验证码或密码），个人薪资趋势图数据
5. **账号安全**：修改密码（需验证旧密码），绑定/修改/解绑手机号（手机号为登录账号，解绑需先绑定新手机号），查看登录日志

### 3.2 功能模块树

```
个人中心
├── 我的档案
│   ├── 查看个人基础信息（敏感字段脱敏）
│   ├── 编辑可修改字段（邮箱/现居住地址/紧急联系人等）
│   └── 锁定字段展示（部门/职位/职级/薪资，提示联系HR）
├── 我的考勤
│   ├── 考勤日历视图（月视图，标记每日状态）
│   ├── 网页端打卡（上班/下班，可配置开关）
│   ├── 补卡申请入口（针对缺卡日期，点击后携带日期参数跳转至考勤模块的补卡申请页，具体补卡审批流由考勤模块+审批中心处理）
│   └── 月考勤统计（出勤天数/迟到/早退/旷工/请假汇总）
├── 我的请假
│   ├── 请假申请列表（按状态筛选）
│   ├── 新建请假申请（跳转考勤模块）
│   ├── 审批进度查看
│   └── 取消申请（仅审批中状态可取消）
├── 我的薪资
│   ├── 历史工资条列表（按月）
│   ├── 工资条详情（二次验证后查看）
│   ├── 薪资趋势图（近6个月实发工资折线图数据）
│   └── 薪资异动通知
└── 账号安全
    ├── 修改密码（验证旧密码 + 新密码复杂度校验）
    ├── 绑定/修改/解绑手机号（短信验证码验证；手机号为登录账号，解绑需先绑定新手机号）
    └── 登录日志（最近20条）
```

### 4.3 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

| **code** | **含义**     | **说明**                     |
| -------- | ------------ | ---------------------------- |
| 0        | 成功         | 请求处理成功                 |
| 400      | 参数错误     | 请求参数校验失败             |
| 401      | 未登录       | token 过期或无效             |
| 403      | 无权限       | 越权访问他人数据             |
| 404      | 资源不存在   | 员工/工资条等记录不存在     |
| 500      | 服务器错误   | 系统内部异常                 |

### 5.1 个人信息查看与编辑流程

```
员工进入"我的档案"
       │
       ▼
┌─────────────────────┐
│ 加载个人基础信息      │
│ - 基础字段直接展示    │
│ - 敏感字段脱敏展示    │
│   (身份证/银行卡等)   │
│ - 锁定字段灰显+提示   │
└─────────┬───────────┘
          │
          │ 点击"编辑"按钮
          ▼
┌─────────────────────┐
│ 进入编辑模式          │
│ - 可编辑字段变为输入框 │
│   (邮箱/地址/紧急联系人)│
│ - 锁定字段保持灰显    │
└─────────┬───────────┘
          │
          │ 修改后点击"保存"
          ▼
┌─────────────────────┐
│ 后端校验              │
│ - 邮箱格式校验        │
│ - 字段长度校验        │
│ - 敏感操作记录审计    │
└─────────┬───────────┘
          │
     ┌────┴────┐
     │         │
  校验通过   校验失败
     │         │
     ▼         ▼
  更新员工   返回错误提示
  档案信息
     │
     ▼
  返回最新档案
```

     │                 │

## 10. 关键技术设计

> 描述个人中心模块涉及的核心技术方案。

### 10.1 敏感字段脱敏

**使用方：** 我的档案（查看个人基础信息）— 身份证号、手机号、银行卡号、紧急联系人电话等敏感字段在后端序列化时统一脱敏，前端不感知原始数据。非本人查看时强制脱敏，本人查看时完整展示但标记为锁定字段。

个人档案中部分字段需在前端展示时脱敏。脱敏规则在后端统一处理，前端不感知原始数据：

| **字段**       | **脱敏规则**              | **示例**              |
| -------------- | ------------------------- | --------------------- |
| 身份证号       | 保留前4位+后4位，中间替换 | 3301**********1234    |
| 手机号         | 保留前3位+后4位，中间替换 | 138****1234           |
| 银行卡号       | 保留后4位                 | **** **** **** 5678   |
| 紧急联系人电话 | 同手机号                  | 139****5678           |

**实现：** `SensitiveDataUtil` 工具类，注解 `@SensitiveData` 标记需脱敏字段，序列化时自动处理。

### 10.2 可编辑字段白名单

**使用方：** 我的档案（编辑可修改字段）— 员工仅可编辑 email/address/emergencyContact/emergencyPhone 四个字段，Service 层白名单校验拒绝越权更新锁定字段（name/phone/idCard/department/position/jobLevel）

`ProfileConfig` 中维护员工可编辑字段白名单：

```java
@ConfigurationProperties(prefix = "profile.editable")
public class ProfileConfig {
    // 员工可编辑字段
    private List<String> employeeFields = Arrays.asList(
        "email", "address", "emergencyContact", "emergencyPhone"
    );
}
```

`updateProfile()` 方法中白名单校验：遍历 DTO 中非空字段，不在白名单中的字段拒绝更新。

### 10.3 工资条二次验证

**使用方：** 我的薪资 — 工资条详情查看（首次查看需短信验证码或密码二次验证）、工资条列表（`hasViewed` 标记控制是否需要验证）

- 验证码 6 位随机数字，Redis 存储，key=`salary:verify:{userId}:{salaryId}`，TTL=5 分钟
- 同一工资条 1 分钟内不可重复发送（Redis key=`salary:verify:limit:{userId}:{salaryId}`，TTL=60s）
- 错误次数限制：同一工资条验证失败 3 次后锁定 15 分钟
- 验证通过后记录 `salary:viewed:{userId}:{salaryId}=true`，TTL=当前登录 session 过期时间，后续查看无需再次验证
- 降级方案：短信服务不可用时，支持输入登录密码作为二次验证

### 10.4 密码安全策略

**使用方：** 账号安全（修改密码）— BCrypt 加密存储、最近3次密码历史校验、修改成功后 token 黑名单强制重新登录

- 密码存储使用 BCrypt 加密
- 维护最近 3 次密码的 hash 历史（`user_password_history` 表），新密码不能与历史重复
- 密码修改成功后：当前 token 加入 Redis 黑名单 → 通知用户 → 记录操作日志
- 密码复杂度要求：长度 ≥ 8 位，必须包含大写字母、小写字母、数字

### 10.5 消息队列设计

**使用方：** 账号安全（密码修改通知 — 密码修改成功后短信/邮件通知用户；手机号变更通知 — 手机号修改/解绑后通知用户）、我的薪资（工资条查看审计 — 详情查看后记录操作日志）

| **消息类型**     | **交换机**          | **路由键**             | **消费者** | **触发时机**         |
| ---------------- | ------------------- | ---------------------- | ---------- | -------------------- |
| 密码修改通知     | hrms.notification   | security.password      | 通知服务   | 密码修改成功后       |
| 手机号变更通知   | hrms.notification   | security.phone         | 通知服务   | 手机号修改成功后     |
| 工资条查看通知   | hrms.audit          | salary.viewed          | 审计服务   | 工资条详情查看后     |

### 10.6 缓存策略

**使用方：** 我的薪资（工资条验证码 — 首次查看时生成并发送；验证码发送频率限制 — 60s 内不可重复发送；工资条已查看标记 — 验证通过后记录，本次登录无需再次验证）、我的考勤（今日打卡标记 — 防止重复打卡）、待办数量（请假审批结果通知 + 新工资条通知 — 红点角标实时更新）

| **缓存内容**         | **缓存位置** | **过期时间**     | **更新策略**                   |
| -------------------- | ------------ | ---------------- | ------------------------------ |
| 工资条验证码         | Redis        | 5 分钟           | 每次查看时生成                 |

### 11.2 数据安全

- **接口级数据隔离**：所有个人中心接口从 token 中提取 userId，仅返回当前用户数据，禁止通过参数指定 userId（防止水平越权）
- **敏感字段脱敏**：身份证号、手机号、银行卡号在后端序列化时统一脱敏
- **工资条二次验证**：首次查看需短信验证码或密码验证，防止工资信息泄露
- **密码修改通知**：密码被修改后短信/邮件通知用户，若非本人操作可联系 HR
- **操作审计**：密码修改、手机号修改、工资条查看等敏感操作记录操作日志
