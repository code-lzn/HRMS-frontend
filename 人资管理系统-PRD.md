# 1. 概述

## 1.1 项目背景

目前公司内部人力资源管理依赖 Excel 和纸质流程，存在数据分散、算薪效率低、审批不透明等问题。本系统旨在建立统一的员工数字化档案，实现入转调离全流程线上化。

## 1.2 目标用户及定位

| **用户角色** | **核心需求** | **使用频率** |
| --- | --- | --- |
| HR 专员 | 员工档案管理、薪资核算、考勤统计 | 每日 |
| 部门主管 | 团队人员情况查看、下属请假审批 | 每日 |
| 普通员工 | 查看个人信息、提交请假、查看工资条 | 每周 |
| 财务专员 | 薪资审核、成本报表 | 每月 |
| 系统管理员 | 权限配置、系统维护 | 低频 |

## 1.3 名词定义

| **名词** | **定义** |
| --- | --- |
| 在职状态 | 员工当前工作状态：试用期 / 正式 / 待离职 / 已离职 |
| 账套 | 薪资计算模板，定义工资构成项目（基本、津贴、扣除等） |
| 考勤组 | 适用相同考勤规则的员工集合 |
| 应发工资 | 税前应发 = 基本 + 津贴 + 绩效 + 加班 |
| 实发工资 | 税后实发 = 应发 - 社保 - 公积金 - 个税 |
| 入转调离 | 入职、转正、调岗、离职四大人事异动 |

# 3. 组织架构管理

## 3.1 部门管理

### 3.1.1 功能定位

维护公司多级组织架构，支持部门增删改查及人员统计。

### 3.1.2 部门字段定义

| **字段** | **必填** | **说明** |
| --- | --- | --- |
| 部门名称 | 是 | 如"技术部" |
| 部门编码 | 是 | 用于工号生成 |
| 上级部门 | 否 | 空表示根部门 |
| 部门负责人 | 否 | 决定数据权限 |
| 排序序号 | 是 | 越小越靠前 |
| 部门描述 | 否 | 职能说明 |

### 3.1.3 部门层级限制

*   最大层级深度：5 级
    
*   每个部门下属子部门数量不限
    
*   部门合并时：被合并部门的员工需先转移至新部门
    

### 3.1.4 部门人数统计

*   实时显示该部门在职员工数（含下属部门）
    
*   统计口径：在职状态为"试用期"或"正式"的员工

# 5. 入转调离流程

## 5.1 入职流程完整设计

### 5.1.1 业务状态流转图

```plain
新建候选人
     │ 填写信息
     ▼
┌──────────┐    保存为草稿      ┌──────────┐
│ 信息录入  │ ─────────────────> │   草稿   │
└─────┬────┘                     └────┬─────┘
      │                                │
      │ 填写完整，提交审批              │ 编辑后提交
      ▼                                ▼
┌──────────────────────────────────────────────────┐
│                   审批中                          │
│  ┌────────────────────────────────────────────┐  │
│  │ 审批流：部门负责人 → [HR负责人]（可选）    │  │
│  └────────────────────────────────────────────┘  │
└─────────────────────────┬────────────────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
        ▼                 ▼                 ▼
   审批通过           审批拒绝           转交他人
        │                 │                 │
        ▼                 ▼                 ▜────> 新审批人
  ┌──────────┐       ┌──────────┐          
  │ 已批准待  │       │  已拒绝   │          
  │   入职   │       │ (可重新)  │          
  └────┬─────┘       └──────────┘          
       │
       │ 实际到岗，HR确认
       ▼
  ┌──────────┐
  │   已入职  │  ← 进入员工档案，开始计算考勤/薪资
  └──────────┘
```

### 5.1.2 状态说明

| **业务状态** | **可见角色** | **可执行操作** |
| --- | --- | --- |
| 草稿 | HR本人 | 编辑、删除、提交审批 |
| 审批中 | HR、当前审批人 | HR可撤回（仅限第一级） |
| 已批准待入职 | HR、部门负责人 | 修改入职日期、标记放弃 |
| 已拒绝 | HR | 查看原因、重新发起 |
| 已入职 | 全员（按权限） | 发起转正流程 |

### 5.1.3 入职申请表单字段

| **字段** | **必填** | **说明** |
| --- | --- | --- |
| 姓名 | 是 |  |
| 性别 | 是 |  |
| 手机号 | 是 |  |
| 邮箱 | 是 |  |
| 身份证号 | 是 |  |
| 预计入职日期 | 是 |  |
| 所属部门 | 是 |  |
| 职位 | 是 |  |
| 录用类型 | 是 | 全职/兼职/实习 |
| 试用期(月) | 是 | 默认取职位配置 |
| 试用期薪资比例 | 是 |  |
| 直接汇报人 | 否 | 默认部门负责人 |

### 5.1.4 审批规则

*   一级审批：部门负责人（固定）
    
*   二级审批：HR负责人（可配置开关）
    
    *   开关条件：非标准职位、或薪资超出职级范围时需二审
        

*   审批时效：每级 48 小时，超时自动升级或催办
    

### 5.1.5 通过后的自动处理

1.  生成工号（规则见 4.1.1）
    
2.  创建系统账号（初始密码随机，首次登录强制修改）
    
3.  发送欢迎邮件给候选人
    
4.  发送确认通知给 HR 和部门负责人

## 5.2 转正流程

### 5.2.1 触发条件

*   系统每天扫描试用期员工
    
*   入职日期 + 试用期（月）- 7 天时，提醒 HR 准备转正
    

### 5.2.2 状态流转

```plain
┌──────────┐
│  试用期  │  ← 员工当前状态
└────┬─────┘
     │ 到达转正评估时间
     ▼
┌──────────┐
│ 待发起   │  ← 系统提醒 HR
│ 转正评估 │
└────┬─────┘
     │ HR发起转正申请
     ▼
┌──────────┐
│ 转正审批 │  ← 流转：部门负责人 → HR负责人
│   中     │
└────┬─────┘
     │
  通过│         拒绝
     ▼            ▼
┌──────────┐  ┌──────────┐
│  已转正   │  │ 延长试用  │
│ (状态变  │  │ or 辞退  │
│  为正式) │  └──────────┘
└──────────┘
```

### 5.2.3 转正申请表单

*   员工信息（系统自动带出）
    
*   试用期开始日期、结束日期
    
*   试用期表现评价：文本，必填
    
*   转正后薪资调整：可选，如调整需审批
    
*   审批结果：通过 / 延长试用 / 不通过
    

---

## 5.3 调岗流程

### 5.3.1 发起条件

*   员工在职状态为"试用期"或"正式"
    

### 5.3.2 可调项

*   所属部门（必须变更）
    
*   职位（可选变更）
    
*   职级（可选变更）
    
*   直接汇报人（可选变更）
    
*   薪资调整（可选，如调整需额外审批）
    

### 5.3.3 审批流

```plain
HR发起调岗申请
      │
      ▼
┌───────────────────┐
│ 原部门负责人审批   │  ← 知情确认
└─────────┬─────────┘
          │ 同意
          ▼
┌───────────────────┐
│ 新部门负责人审批   │  ← 接收确认
└─────────┬─────────┘
          │ 同意
          ▼
┌───────────────────┐
│   HR负责人备案    │  ← 最终确认
└─────────┬─────────┘
          │
          ▼
     调岗生效
```

### 5.3.4 调岗生效处理

*   更新员工档案中的部门、职位等信息
    
*   记录调岗历史（原岗位、新岗位、调岗日期、原因）
    
*   工号不变

## 5.4 离职流程

### 5.4.1 发起条件

*   员工在职状态为"试用期"或"正式"
    
*   由 HR 发起（员工申请离职走另一个申请流程，通过后 HR 发起）
    

### 5.4.2 状态流转

```plain
HR发起离职申请
      │
      ▼
┌──────────────┐
│ 部门负责人审批 │  ← 确认交接安排
└──────┬───────┘
       │ 同意
       ▼
┌──────────────┐
│  HR负责人审批  │  ← 最终确认
└──────┬───────┘
       │
       ▼
  审批通过
       │
       ▼
┌──────────────┐
│   待离职      │  ← 等待最后工作日
│ (状态变更)    │
└──────┬───────┘
       │ 到达离职日期
       ▼
┌──────────────┐
│   已离职      │  ← 账号禁用，档案保留
└──────────────┘
```

### 5.4.3 离职申请表单

*   员工信息（带出）
    
*   离职日期：必须 >= 今天
    
*   离职原因：下拉选择（主动/被动/协商）+ 详细说明
    
*   离职类型：辞职/辞退/合同到期不续签/其他
    
*   工作交接人：必须指定
    

### 5.4.4 离职生效处理

*   员工状态变为"已离职"
    
*   系统账号禁用（无法登录）
    
*   工号标记释放（本年度可复用）
    
*   从考勤组移除
    
*   薪资计算至离职日期
    
*   保留员工档案可查询（脱敏敏感信息）

# 6. 考勤管理

## 6.1 考勤规则配置

### 6.1.1 考勤组定义

考勤组是一组适用相同考勤规则的员工集合。

| **字段** | **必填** | **说明** |
| --- | --- | --- |
| 考勤组名称 | 是 | 如"标准工时组"、"弹性工时组" |

## 6.3 请假管理

### 6.3.1 请假类型

| **类型** | **说明** | **余额管理** | **证明材料** |
| --- | --- | --- | --- |
| 年假 | 法定+公司福利 | 是，按入职时长计算 | 无需 |
| 病假 | 因病休假 | 否 | 医院证明（>1 天） |
| 事假 | 因事请假 | 否 | 无需 |
| 婚假 | 结婚休假 | 否 | 结婚证复印件 |
| 产假 | 生育休假 | 否 | 医院证明 |
| 丧假 | 直系亲属丧葬 | 否 | 无需 |
| 调休 | 加班调休 | 是，需有余额 | 无需 |

### 6.3.2 假期余额计算

**年假公式：**

*   入职满 1 年不满 10 年：5 天/年
    
*   入职满 10 年不满 20 年：10 天/年
    
*   入职满 20 年：15 天/年
    
*   当年入职按剩余月份折算：加入月份至年末月份数 / 12 × 对应天数
    

**调休公式：**

*   加班时长累计，1:1 转换为调休时长
    
*   调休有效期：加班当月及次月有效，过期清零
    

### 6.3.3 请假申请

| **字段** | **必填** | **说明** |
| --- | --- | --- |
| 请假类型 | 是 | 下拉选择 |
| 开始时间 | 是 | 日期+时段（上午/下午） |
| 结束时间 | 是 | 日期+时段 |
| 请假天数 | 系统计算 | 支持 0.5 天 |
| 请假事由 | 是 | 文本描述 |
| 工作交接 | 否 | 指定交接人 |
| 附件 | 条件必填 | 病假/婚假/产假需上传证明 |

### 6.3.4 审批流规则

| **请假类型+天数** | **审批人** | **备注** |
| --- | --- | --- |
| 年假/调休 ≤ 3 天 | 直接上级 | \- |
| 年假/调休 > 3 天 | 直接上级 → 部门负责人 | \- |
| 病假/事假 ≤ 1 天 | 直接上级 | \- |
| 病假/事假 > 1 天 | 直接上级 → 部门负责人 | \- |
| 婚假/产假/丧假 | 直接上级 → HR 备案 | 无需二审 |

## 6.4 考勤统计

### 6.4.1 统计维度（个人维度）

*   应出勤天数：当月工作日
    
*   实际出勤天数：有打卡记录的天数

## 7.1 薪资账套（工资模板）

### 7.1.1 账套定义

账套是薪资计算的基础模板，定义工资构成项目和计算规则。

| **字段** | **必填** | **说明** |

| 生效日期 | 是 | 账套启用日期 |
| 工资项目 | 是 | 配置多个项目 |

### 7.1.2 工资项目类型

| **类型** | **说明** | **计算方式** | **示例** |

| 考勤扣款 | 根据考勤数据 | 规则计算 | 迟到扣款、请假扣款 |
| 社保扣除 | 社保个人部分 | 基数×比例 | 养老、医疗、失业 |
| 公积金扣除 | 公积金个人部分 | 基数×比例 | 公积金 |
| 个税 | 个人所得税 | 累计预扣法 | 个税 |

## 7.3 月度薪资核算

### 7.3.1 核算流程

```plain

         ▼

└──────────────────┬───────────────────┘
                   │
                   ▼
┌──────────────────────────────────────┐
│ 自动计算                              │
│ 遍历每位员工：                        │
│ 1. 取薪资档案                         │
│ 2. 取考勤数据计算扣款                 │
│ 3. 应用账套公式逐项计算               │

                   │
                   ▼

└──────────────────┬───────────────────┘
                   │
              ┌────┴────┐
              │         │
         确认无误     需要调整
              │         │
              ▼         ▼
         提交审批    添加调整项

              │
              ▼
```

### 7.3.2 薪资批次状态

| **状态** | **说明** | **可执行操作** |
| --- | --- | --- |

| 待确认 | 计算完成，等待 HR 确认 | 查看预览、调整、提交审批 |
| 审批中 | 已提交财务审批 | 查看进度 |
| 已通过 | 审批通过，工资条可见 | 发放确认 |
| 已发放 | 实际工资已发放 | 归档 |
| 已驳回 | 审批未通过 | 修改后重新提交 |

### 7.3.3 异常检测规则

系统计算完成后自动标记异常：

*   当月请假天数 > 15 天 → 黄色预警
    
*   当月加班费 > 50 小时 → 黄色预警
    
*   薪资较上月变动 > 30% → 红色预警（需确认）
    
*   新入职员工未设置薪资档案 → 红色阻断（无法计算）
